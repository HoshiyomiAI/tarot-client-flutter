# UI 与主题约定（UI & Theme）

本模板提供轻量主题与基础组件结构，建议你在此基础上沉淀通用 UI。

## 主题（Theme）
- 主题入口位于 `lib/app.dart`，统一颜色、字号与组件风格
- 推荐支持明暗主题切换，并将关键色值抽取为常量

## 页面骨架（Screens）
- 每个页面对应一个 `Screen` 组件，保留基础 `AppBar` 与内容区
- 页面文件放在 `lib/screens/`，命名规范：`<feature>_screen.dart`

## 组件（Widgets）
- 通用组件置于 `lib/widgets/`，命名规范：`<purpose>_<type>.dart`
- 组件拆分原则：复用性、单一职责、避免过重的状态耦合

## 交互与动画
- 保持用户可预期的导航与返回行为
- 对列表与转场的动画建议封装到独立 `Widget`

## 可访问性（A11y）
- 文本大小与对比度：遵循基础可读性标准
- 触控区域：尽量不小于 40x40 dp

## 代码规范与检查
- 空安全与分析：参考 `analysis_options.yaml`
- 命名与文件组织：模块聚合、避免过度嵌套
